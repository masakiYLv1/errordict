# ErrorDict 要件定義書

## 1. プロジェクト目的

- プログラミング学習中に遭遇したエラーとその解決方法を整理・蓄積できる Web アプリを作成する
- Vite/React/TypeScript/Supabase/Chakra UI の技術を組み合わせた開発経験
- 個人開発を通じて、設計からデプロイまで一連の開発フローを経験する

## 2. 想定ユーザー

- 過去に遭遇したエラーと解決方法を自分用に記録・検索したい人
- ブログ記事にて転用したい人（Qiita / Zenn など）

## 3. 成功の基準

- **Phase1(MVP)**
  - エラーの一覧表示・キーワード（タイトル）検索
  - エラーの登録・編集・削除
- **Phase2**
  - 認証・権限制御が正しく機能している

## 4. 主要機能

- **Phase1(MVP)**
  - エラー一覧表示・詳細表示
  - エラーの登録・編集フォーム
  - エラーの削除
- **Phase2**
  - 未ログインユーザーは一覧・詳細のみ閲覧可能
  - 登録・編集・削除はログインユーザーのみ可能（RLS で制御）

## 5. 非機能要件

- ローディング画面の実装
- エラー登録・更新時は成功/失敗をユーザーに通知する

## 6. 管理するエラー情報

- エラータイトル
- エラーメッセージ
- 発生した技術（React / TypeScript / Supabase など）
- 原因
- 解決方法

## 7. 制約事項

- 開発期間：1 ヵ月程度
- 開発者：1 名
- 生成 AI
  - AI（Open Codex など）の使用は極力控えること（最後の手段として使用すること）**\*1**
  - VSCode の GitHub Copilot の機能は使用しないこと

**\*1 コードやドキュメントの最終評価は Open Codex を使用**
